# 大渡河AI — 前端接口文档

> 面向: 前端开发 · 版本: v0.1.0 · 更新: 2026-06-27

---

## 架构说明

前端就是一个聊天窗口，所有复杂逻辑由后端 agent-reasoning 服务编排。前端只需调一个接口。

```
浏览器 ──POST /api/v1/chat──→ agent-reasoning:8003
                                    │
                                    ├── 意图识别 (LLM)
                                    ├── 检索 (调 search-engine:8002)
                                    ├── 知识图谱查询 (调 graph-engine:8001)
                                    ├── 答案生成 (LLM)
                                    └── 引用组装
```

---

## 1. 通用约定

### 响应体格式

```json
{
  "code": 0,
  "message": "ok",
  "trace_id": "fxl-a1b2c3d4-...",
  "data": { ... }
}
```

- `code` = 0 表示成功，非 0 失败
- `trace_id` 排查问题时提供给后端

### 请求头

| Header | 必填 | 说明 |
|--------|:----:|------|
| `Content-Type: application/json` | 是 | — |
| `X-Trace-Id` | 否 | 格式 `web-{uuid}`，不传则服务端自动生成 |

## 2. 智能问答（唯一前端接口）

```
POST /api/v1/chat
Content-Type: application/json
```

Base URL: `http://{host}:8003`

### 请求

```json
{
  "query": "混凝土坝裂缝宽度超过多少需要处理？",
  "conversation_id": "conv-a1b2c3d4"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|:--:|------|
| `query` | string | 是 | 用户输入的问题 |
| `conversation_id` | string | 否 | 会话 ID。**首次对话不传**，后续用接口返回的值 |

### 响应 (HTTP 200)

### answer 渲染规则

answer 文本中的 `[1]` `[2]` 是引用标记，前端需将其替换为可点击的引用角标：

```
正则匹配: \[(\d+)\]
映射关系: [1] → citations 数组中 index=1 的那条
```

渲染效果：

```
根据《混凝土坝安全监测技术规范》DL/T 2628-2023 第5.2.3条规定，
当裂缝宽度大于0.3mm时，应采取灌浆或表面封闭处理措施。¹

此外，《水工建筑物缺陷管理规范》第6.1.2条进一步将裂缝宽度>0.3mm
判定为较大缺陷，建议优先采用灌浆法处理。²
```

点击角标 `¹` 展开引用卡片：


### 流式输出 (SSE)

设置 `"stream": true` 后，响应切换为 SSE (`text/event-stream`) 格式，不再返回 JSON。

**请求：**

```json
{
  "query": "混凝土坝裂缝宽度超过多少需要处理？",
  "conversation_id": "conv-xxx",
  "stream": true
}
```

**SSE 事件类型：**

| event | 出现时机 | data 内容 |
|-------|---------|----------|
| `status` | 意图识别 / 检索完成 | `{"node":"supervisor","query_type":"knowledge_qa","sub_questions":1}` |
| `answer_chunk` | LLM 逐 token 输出 | `{"token":"混凝土"}` — 前端按顺序拼接 |
| `references` | 答案输出完成 | citations 数组（与上面 `data.citations` 同结构） |
| `done` | 整个请求结束 | `{"conversation_id":"conv-xxx","trace_id":"...","elapsed_ms":5175}` |
| `error` | 出错 | `{"error":"错误信息"}` |

**前端示例：**

```typescript
async function* chatStream(query: string, convId: string | null): AsyncGenerator<{
  type: 'status' | 'answer_chunk' | 'references' | 'done' | 'error';
  data: any;
}> {
  const resp = await fetch('http://{host}:8003/api/v1/chat', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ query, conversation_id: convId, stream: true }),
  });

  const reader = resp.body!.getReader();
  const decoder = new TextDecoder();
  let buffer = '';

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;
    buffer += decoder.decode(value, { stream: true });

    // 解析完整的 SSE 行
    const lines = buffer.split('\n');
    buffer = lines.pop() || '';

    let eventType = '';
    for (const line of lines) {
      if (line.startsWith('event: ')) {
        eventType = line.slice(7).trim();
      } else if (line.startsWith('data: ')) {
        yield { type: eventType as any, data: JSON.parse(line.slice(6)) };
      }
    }
  }
}

// 使用
let answer = '';
for await (const evt of chatStream('混凝土坝裂缝怎么处理', convId)) {
  switch (evt.type) {
    case 'answer_chunk':
      answer += evt.data.token;
      // 实时追加到 DOM
      break;
    case 'references':
      // 渲染引用角标
      break;
    case 'done':
      convId = evt.data.conversation_id;
      break;
    case 'error':
      console.error(evt.data.error);
      break;
  }
}
```

### 多轮对话

```typescript
let conversationId: string | null = null;

async function chat(query: string) {
  const resp = await fetch('http://{host}:8003/api/v1/chat', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      query,
      conversation_id: conversationId,  // null 时后端创建新会话
    }),
  });
  const json = await resp.json();
  conversationId = json.data.conversation_id;  // 保存，下轮继续用
  return json.data;
}
```

## 3. 文档下载

答案中引用的文档下载链接通过 doc-parser 获取，前端直接作为 `<a>` 的 href：

```
GET /api/v1/documents/{doc_id}/download
```

Base URL: `http://{host}:8080`

响应为 HTTP 302 重定向到 MinIO 预签名 URL，浏览器自动触发下载。**前端不需要额外处理。**

## 4. 错误处理

### 错误响应格式

```json
{
  "code": 4001,
  "message": "upstream service unavailable: search-engine",
  "trace_id": "fxl-a1b2c3d4-..."
}
```

### 常见错误码

| code | 含义 | 前端处理 |
|------|------|---------|
| `0` | 成功 | — |
| `1002` | 参数错误 | toast message 内容 |
| `4001` | 后端服务不可用 | 提示"服务繁忙，请稍后重试" |
| `5005` | LLM 调用失败 | 提示"AI 暂时无法响应" |
| `9001` | 系统内部错误 | 提示"系统异常"，记 trace_id |
