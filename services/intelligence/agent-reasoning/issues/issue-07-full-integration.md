# Issue 7: Chat Endpoint + Full Integration — 全链路集成

## Type

AFK

## Blocked by

- Issue #5 (Context Resolution + Multi-Turn)
- Issue #6 (LLM Factory)

## What to build

将所有节点串联为完整的 LangGraph StateGraph，接线 `POST /api/v1/chat` 端点，端到端验证全链路。

**1. 完整 Graph 拓扑**（`src/graph/builder.py` 重写）

```
supervisor ──┬── query_type=chitchat ────→ generator → END
             │
             └── query_type≠chitchat ──→ context_resolution → call_tools → fusion → generator ──┬── chitchat → END
                                                                                                   │
                                                                                                   └── 非chitchat → citation → END
```

6 个节点 + 2 处条件路由：
- supervisor 之后：chitchat → generator，非 chitchat → context_resolution
- generator 之后：chitchat → END（跳过 citation），非 chitchat → citation → END

**2. POST /api/v1/chat 端点**（`src/main.py` 更新）

- 解析 ChatRequest，校验参数（query、retrieval_mode）
- 生成 trace_id（透传或自动生成）
- 注入 ConversationStore 中已有的 conversation_id（或新建）
- 调用 `_graph.ainvoke(state)`
- 请求完成后，将本轮 user query 和 assistant answer 追加到 ConversationStore
- 返回 ChatResponse（code=0, answer, citations, trace_id 等）
- 错误处理：
  - Pydantic 校验失败 → 422, code=1002
  - Graph 执行异常 → 500, code=9001
  - Supervisor 失败（不可降级）→ 500, code=9001

**3. 全链路集成测试**

使用真实 search-engine + 真实本地 LLM，覆盖 Phase A 全部场景。

## Acceptance criteria

- [ ] 完整的 6 节点 Graph 编译成功
- [ ] 条件路由正确：chitchat 走快捷路径，其他走完整路径
- [ ] POST /api/v1/chat 返回 code=0，含 answer 和 citations
- [ ] 全链路 trace_id 贯穿（从 HTTP header 到日志到响应体）
- [ ] 多轮对话：第二轮能正确消解前一轮的引用
- [ ] 部分检索失败不中断，有结果就继续
- [ ] 检索无结果时 answer 为"未找到"风格回复
- [ ] 所有 4 种 query_type 场景端到端通过
- [ ] 单问题知识问答（Q1-1 ~ Q1-5）：正确 answer + 来自真实数据的 citations
- [ ] 复合问题（Q2-1 ~ Q2-4）：2+ 子问题正确分解，综合回答覆盖全部维度
- [ ] 对比问题（Q3-1 ~ Q3-4）：comparison 类型 + 对比性回答
- [ ] 多轮对话（Q4-1 ~ Q4-5）：第二轮指代消解正确，引用前一轮上下文
- [ ] 规范条文查询（Q5-1 ~ Q5-5）：spec_lookup 类型 + 规范引用风格
- [ ] 多跳推理（Q6-1 ~ Q6-3）：串联多 chunk 信息，回答具备推理链
- [ ] 检索无结果（Q7-1 ~ Q7-4）："未找到"风格回复，不编造
- [ ] Chitchat（Q8-1 ~ Q8-4）：友好回复，无检索 citations
- [ ] 错误处理（Q9-1 ~ Q9-5）：正确错误码 + trace_id
