## 2. 启动四个服务（四个终端，按依赖顺序）

工作目录统一为 monorepo 根：

```bash
cd /mnt/data_nvme/code/Daduhe-AI/services/intelligence
```

### 终端 1: llm-gateway (8004) — LLM 调用工厂 + 缓存

```bash
cd llm-gateway && uv sync && uv run uvicorn src.main:app --host 0.0.0.0 --reload --port 8004
```

其他三个服务都依赖它，所以**最先启**。

### 终端 2: graph-engine (8001) — 知识图谱查询

```bash
cd graph-engine && uv sync && uv run uvicorn src.main:app --host 0.0.0.0 --reload --port 8001
```

### 终端 3: search-engine (8002) — 多模式检索

```bash
cd search-engine && uv sync && uv run uvicorn src.main:app --host 0.0.0.0 --reload --port 8002
```

### 终端 4: agent-reasoning (8003) — 核心入口（前端对接）

```bash
cd agent-reasoning && uv sync && uv run uvicorn src.main:app --host 0.0.0.0 --reload --port 8003
```

> `--host 0.0.0.0` 使服务监听所有网卡，局域网内的前端开发机可直接访问。
> `--reload` 表示代码改动自动重启，开发方便。
>
> **启动顺序说明**: llm-gateway 必须先启（其他服务内部 LLM 调用都走它）。graph-engine / search-engine 无先后依赖，可并行启动。agent-reasoning 最后启（依赖前三个）。

## 3. 验证

```bash
# 健康检查
curl http://localhost:8003/health

# 非流式问答
curl -s http://localhost:8003/api/v1/chat \
  -H "Content-Type: application/json" \
  -d '{"query": "你好"}'

# 流式 SSE 问答（-N 禁止缓冲，实时查看）
curl -s -N http://localhost:8003/api/v1/chat \
  -H "Content-Type: application/json" \
  -d '{"query": "混凝土坝裂缝宽度超过多少需要处理", "stream": true}'

# 流式测试脚本（打字机效果）
cd agent-reasoning && uv run python tests/test_sse_stream.py "混凝土裂缝的分类标准有哪些" --speed 30
```

前端开发时只需指向 `http://<本机IP>:8003`。

---

## 4. 为什么不用 Docker

Docker Compose 会启动全套基础设施（PG/Milvus/Memgraph/Ollama）的本地容器，但本地没有 GPU 跑不动 Ollama，也没有 DeepSeek API Key 配在环境变量里。当前代码默认配置已经指向远程服务器，直接用 uv 最省事。
